# FAQ

## What is this repository?

This repository is a public-facing package for staged validation of a hydrogen-storage MOF candidate with an interesting ambient-temperature and 100 bar modeled signal.

## Is the candidate experimentally proven?

No. The current evidence is computational and method-documented, but external physical validation has not happened yet.

## Why publish before the lab work is done?

Because the project already has enough structure to be reviewed seriously. Publishing the workflow, evidence boundary, and validation plan makes it easier for labs, collaborators, and funders to understand the next step.

## What is the next real milestone?

The next major milestone is obtaining a real synthesis quote from a qualified lab and determining whether the candidate or a close equivalent can be made under realistic conditions.

## Why include negative-result language?

Because a disciplined negative result is still scientifically useful. It helps document the gap between modeled candidates and experimental reality.

## Is this a request for a large research grant?

No. The project is intentionally framed as a staged validation effort with small, decision-oriented milestones.

## What kind of help is most useful right now?

The most useful help is:

- a synthesis quote or technical fit review
- a referral to a qualified lab or academic group
- modest milestone-based support for first-pass validation

## Why keep the repository documentation-focused?

At this stage, documentation does the main job: it shows the project is organized, bounded, and honest about what is and is not proven.
