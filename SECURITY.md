# Security Policy

## Scope

This project may include code, documentation, and public research materials.

Security concerns can include:

- accidental exposure of credentials or tokens
- unsafe publication of sensitive environment details
- malicious code submissions
- unsafe operational guidance embedded in scripts or docs

## Reporting

If you find a security or sensitive-data issue, do not post secrets publicly in an issue.

Instead, report the problem privately to the repository maintainer using the contact method that will be added when the public repository is created.

## Public Safety Note

If a concern involves dangerous lab misuse rather than software security, treat it as a safety issue and report it with clear context.
